Погнали!

Постановка задачи

POST /rating
{
  "companyId": UUID,
  "value": int
}

value < 100        — Низкий
101 ≤ value ≤ 200  — Средний
value > 201        — Высокий

Вот задание: эндпоинт POST /rating принимает companyId в формате UUID и value — целое число. Рейтинг классифицируется как Низкий, Средний или Высокий в зависимости от value. Нужно составить тест-план и примеры проверок. Проблема? Это вся информация! Нет деталей о формате ответа, ошибках, авторизации или бизнес-логике. Почему задание такое сырое? Начнем с того, чего хотят интервьюеры.
